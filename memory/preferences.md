# Preferencias de colaboración

## Estilo de código
- Sin comentarios innecesarios (solo el "por qué" no obvio)
- Sin abstracciones prematuras; no diseñar para requisitos hipotéticos futuros
- Sin manejo de errores para escenarios imposibles
- Preferir editar archivos existentes a crear nuevos

## Comunicación
- Respuestas cortas y directas
- UI y mensajes al usuario siempre en español
- Código y nombres de variables en inglés
- Sin emojis salvo que se pidan explícitamente

## Diseño
- Mobile-first (max-width 430px)
- Sin librerías UI externas; componentes propios en `components/ui/`
- Currency: COP, locale es-CO

## Git
- No hacer commit salvo que se pida explícitamente
- No hacer push salvo confirmación explícita
